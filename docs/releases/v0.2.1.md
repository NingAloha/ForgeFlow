# ForgeFlow v0.2.1 — Stabilization Release Checklist

## Version Identity

v0.2.1 is a stabilization release focused on governed materialization reliability, attempt lifecycle semantics, and replay/gate observability. No mutation semantics are introduced in this release.

## PR1（failure semantics）— 已完成

Status: merged / verified on `main`

- Code 已落地：`started`=incomplete blocker，`failed`=signal only（retry allowed），`completed`=terminal success（rerun no-op）
- Tests 已覆盖：started/failed/completed 的分支、retry/no-op、events、replay rendering、gate snapshot
- Replay/Gate 已接入：replay 可解释 `Materialization: status`；gate 对 `started` 阻断、对 `failed` 不阻断
- CLI no-op 文案已存在：`Materialization preview already completed; no-op.`

## PR2（幂等重试固定化）— 已完成

Status: merged / verified on `main`

Branch: `track/materialization-idempotent-retry`（仅当需要单独 PR/记录验证时）

Non-goal:
- No retry scheduler
- No background recovery loop
- No automatic resume semantics

验收（逐条跑通并记录结果）：
- `failed` 再次 `--materialize-preview` 必须 retry（不应 no-op）
- `completed` 再次 `--materialize-preview` 必须 no-op，且文案精确匹配：`Materialization preview already completed; no-op.`
- gate：`failed` 不阻断，`started` 阻断
- replay 稳定显示：
  - `Materialization:`
  - `- status: failed|completed|started`

## PR3（Golden / docs 收口）— 已完成

Status: merged / verified on `main`

Branch: `docs/materialization-semantics`

- 更新 `docs/runtime-v0.2-walkthrough.md`
- 增加一节：**Materialization attempt lifecycle**
  - `started` 是异常未闭合（incomplete attempt blocker）
  - `failed` 是可恢复失败（signal only, retry allowed）
  - `completed` 是终局成功（terminal success, rerun no-op）
  - materialization preview 是 preview write-path（governed），不是 mutation 语义（不引入执行/patch apply/源码修改）

## v0.2.1 Definition of Done（最终门槛）

自动化：
- `ruff check .`
- `pytest -q`

手动稳定性回归（同一个 `<run_id>`/`<state_dir>`）：
- `python3.11 main.py --materialize-preview --run-id <run_id> --state-dir <state_dir>`（第一次应 completed）

- `python3.11 main.py --execution-gate --run-id <run_id> --state-dir <state_dir>`（completed 不阻断；started 才阻断）
