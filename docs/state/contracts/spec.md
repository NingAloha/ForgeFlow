# `spec.json`

用于保存需求阶段产物，只回答“要做什么”“给谁做”“做到什么算完成”，不讨论实现细节。

```json
{
  "project_goal": "",
  "target_users": [],
  "functional_requirements": [],
  "non_functional_requirements": [],
  "constraints": [],
  "preferences": [],
  "acceptance_criteria": [],
  "open_questions": []
}
```

字段说明：

* `project_goal`：项目最终目标
* `target_users`：目标用户或使用者群体
* `functional_requirements`：系统必须提供的能力
* `non_functional_requirements`：性能、稳定性、可维护性、交互体验等要求
* `constraints`：明确约束
* `preferences`：偏好项
* `acceptance_criteria`：验收标准
* `open_questions`：当前还未澄清的问题
